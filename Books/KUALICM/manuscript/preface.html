Preface

Who should read this book?

This book is intended for Configuration Managers that need to manage releases, quality, automation, and environments for development. This includes managers of Continuous Integration, environment configuration maintenance, and configuration different implementations of Rice.

There are already many experienced and knowledgable professionals at the Kuali Foundation member institutions and affiliates within the community. However, the community is growing and with it new members arrive that can make use of all that these volumes have to offer.

About this Book

A cohesive collection of best practices that are not just intended as suggestions or "This is how I did it", but comprehensively a tried methodology for accomplishing the goal. The goal is developing Kuali ™ software plain and simple. This book isn't really a reference manual. I can see it being used that way, but its main intent and purpose is as a learning manual and educational literature for helping people in specific roles adapt to Kuali software development. Far better references can be found at Kuali's Wiki.
